REST API Reference
Full OpenAPI-compatible REST API. All endpoints accept JSON and return JSON. Authenticate with your Bearer token in the Authorization header.
BASE URL
https://api.vortexdl.app/v1

Download Endpoints
POST /v1/download Submit a new download job. Returns job_id.
GET /v1/jobs List all jobs. Filter by status, agent, platform.
GET /v1/jobs/:id Get single job status, progress, and metadata.
DEL /v1/jobs/:id Cancel and delete a pending or active job.
PATCH /v1/jobs/:id/priority Update job priority. Body: {"priority":"high"}
POST /v1/download — Request & Response
REQUEST BODY
// All fields except url are optional { "url": "https://youtube.com/…", // required "quality": "1080p", // best|4k|1080p|720p|audio "format": "mp4", // mp4|webm|mkv|mp3|m4a "threads": 8, // 4|8|16|32 "priority": "normal", // normal|high|critical "tags": ["tutorial"], // custom labels "cms_push": "contentful", // CMS target slug "callback": "https://…", // webhook on completion "metadata": true, // extract title/desc/captions "agent_id": "agt_yt_01" // attribute to agent }
RESPONSE 202
{ "job_id": "job_d4f8a2c1e9", "status": "queued", "position": 3, "eta_seconds":47, "poll_url": "/v1/jobs/job_d4f8a2c1e9", "created_at": "2026-03-02T14:22:11Z" }
Agent Endpoints
GET /v1/agents List all configured agents and their live status.
POST /v1/agents Create and deploy a new harvesting agent.
PATCH /v1/agents/:id Update agent config (source, interval, target).
POST /v1/agents/:id/run Manually trigger an immediate agent run cycle.
POST /v1/agents/:id/pause Pause/resume agent polling without deleting it.
DEL /v1/agents/:id Remove agent and optionally cancel its pending jobs.
Webhook Endpoints
GET/v1/webhooksList all configured outgoing webhooks.
POST/v1/webhooksRegister a new outgoing webhook URL and event.
DEL/v1/webhooks/:idRemove a webhook registration.
POST/v1/webhooks/:id/testFire a test payload to verify your endpoint.
Harvest / Bulk Endpoints
POST/v1/harvest/playlistQueue all videos in a playlist or channel.
POST/v1/harvest/keywordsSearch and harvest by keyword across platforms.
POST/v1/harvest/batchSubmit array of URLs as a single batch job.
GET/v1/harvest/:id/statusProgress of a batch harvest operation.
CMS Integration Endpoints

POST/v1/cms/connectionsAdd a new CMS connection (WordPress, Contentful, etc).
POST/v1/cms/publish/:job_idManually push a completed job to connected CMS.
GET/v1/cms/mappingsGet field mapping schemas for each CMS target.
System Endpoints

POST/v1/apikeysGenerate additional API keys with scoped permissions.

🔧 n8n Setup Guide
1
Add HTTP Request Node
In n8n, add an HTTP Request node. Set Method to POST and URL to your VortexDL API endpoint.
2
Set Authentication Header
Under Headers, add Authorization: Bearer vdl_live_sk_… using your API key from Settings.
3
Configure Webhook Callback
Add a Webhook Trigger node in n8n. Copy its URL and paste it in the "callback" field of your API request body.
4
Map Response Data
When the download completes, VortexDL POSTs to your callback with job_id, file_url, metadata, and cms_entry_id.
📄 n8n HTTP Request Node — JSON Config
{ "nodes": [ { "name": "VortexDL — Submit Download", "type": "n8n-nodes-base.httpRequest", "parameters": { "method": "POST", "url": "https://api.vortexdl.app/v1/download", "authentication": "headerAuth", "headerParameters": { "parameters": [{ "name": "Authorization", "value": "Bearer vdl_live_sk_{{$credentials.apiKey}}" }] }, "jsonParameters": true, "bodyParametersJson": "={ \"url\": \"{{$json.video_url}}\", \"quality\": \"1080p\", \"format\": \"mp4\", \"callback\": \"{{$webhookUrl}}\", \"cms_push\": \"contentful\" }" } }, { "name": "Wait for Completion Callback", "type": "n8n-nodes-base.webhook", "parameters": { "path": "vortexdl-done", "responseMode": "responseNode" } } ] }
Tray.io Connector Setup
1
Use the HTTP Client Service
In Tray.io, add a HTTP Client step. Configure the base URL as https://api.vortexdl.app/v1.
2
Add Authentication
Under Authentication, choose Header-based and add your Bearer token from the VortexDL API Settings panel.
3
Configure Trigger
Use the Tray.io Webhook Trigger to receive completion callbacks and feed data into downstream CMS or storage connectors.
Make.com (Integromat) Setup
1
Add HTTP → Make a Request Module
In Make.com, search for HTTP → Make a request and set Method to POST. Enter the VortexDL API endpoint URL.
2
Set Headers and Body
Add the Authorization header and configure a JSON body with url, quality, and callback fields.
3
Use Webhooks Module for Callback
Add a Webhooks → Custom webhook module. Copy the URL and use it as the callback in your download request.
Zapier Webhook Setup
1
Create a Webhooks by Zapier Action
In your Zap, choose Webhooks by Zapier → POST. Set the URL to https://api.vortexdl.app/v1/download.
2
Add Header Authentication
Under Headers, add Authorization: Bearer vdl_live_sk_… and set Content-Type to application/json.
3
Use Catch Hook for Callbacks
Add a Webhooks by Zapier → Catch Hook trigger Zap. Use that URL as the callback to receive download completion data.
